# 微信游戏社区「定向社交」功能方案

## 一、需求背景和目的

从社区增长的角度看，核心围绕拉新、留存、促活三件事。这个功能在这三个方向上都有明确的作用点：

**拉新：通过用户的社交行为把新人带进来。**

用户在社区内@好友或提醒好友来看某条帖子，等于用社交关系完成了一次天然的邀请。被@的人如果此前不活跃甚至没进过社区，这就是一次零成本的拉新——动机来自真实的人际关系，而非平台的推送或广告，转化效率会更高。

**留存：让用户有持续回来的理由。**

用户留下来的前提是这里有人跟他互动。如果社区里有人@他、提到他、给他定向分享内容，他就有了一个具体的回来看看的理由。这种"有人在找你"的感知，比被动刷内容的留存效果强得多。对于那些装了游戏但很少打开社区的沉默用户来说，一条"XX提到了你"的通知，可能就是让他重新回来的触发点。

**促活：降低发帖门槛，提升互动频次。**

目前社区中用户发帖意愿不高，一部分原因是互动缺失——发了之后没人看、没人回应，几次之后就没动力继续了。"提醒谁看"和"@"的能力，让用户发帖时就能确保特定的人会看到并回应，解决的是"发了也没人理"的问题。

另一个阻碍发帖的因素是隐私顾虑。目前有大量用户不愿意授权游戏数据，也不愿意在社区公开发帖——不确定内容会被谁看到，在绑定微信身份的环境下尤其谨慎。"指定可见范围"的能力直接回应这个顾虑：用户可以明确控制谁能看到自己发的东西，心理安全边界是清晰的，发帖的门槛就降下来了。

这两个层面结合起来：门槛降低→愿意发→发了有人看→有回应→更愿意发。活跃度的提升不是靠单点功能推动的，而是靠这个循环转起来。

## 二、功能说明

我们的方案包含三个能力，各自解决不同环节的问题。

### 第一层：提醒谁看（轻量定向触达）

**解决的问题**：发了但没人看到。

用户发帖时可以选择"提醒"特定的人来看。被提醒的人会在feed中优先看到这条帖子，并有"提到了我"的浅色标记。帖子正文中不会出现任何@文字，发帖者不需要把这种行为公开展示给所有人。

对应的用户场景：
- 分享了一个游戏高光，想让昨天一起打的队友一定看到
- 写了个攻略，想确保那个一直问出装的朋友注意到
- 发了一个新游戏体验，想让可能感兴趣的朋友不要错过
- 不想@得太刻意，但希望对方不会漏掉

为什么采用"提到"而非@的形式：心理负担更低，用户更愿意频繁使用；不暴露社交关系给其他人，只有被提到的人自己知道；跟微信朋友圈"提醒谁看"的心智一致，学习成本几乎为零。从粘性角度看，越轻的功能使用频次越高，用户越容易形成习惯。

被提到方的视角：
- feed中看到帖子时多一行浅灰色"提到了我"
- 朋友Tab入口处出现发帖人头像提示
- 消息页收到一条"XX分享了一条动态给你"的通知

---

### 第二层：@艾特（显性社交行为）

**解决的问题**：需要公开的、强烈的定向表达。

用户在帖子正文或评论中显性@某人，被@的人名以蓝色高亮文字出现在内容中，所有能看到帖子的人都知道你@了谁。

对应的用户场景：
- 五杀翻盘发帖"@队友1 @队友2 还记得那波团吗"——公开纪念、带仪式感
- 讨论帖里@某个大神来回答问题
- 在评论区@朋友让他来看这条帖子
- 需要做一个公开的喊话，让被@的人和围观的人都看到

社区是开放环境，@是社区的基础能力，用户对此有明确预期。在评论区的@场景尤其重要——用户可以在任何一条帖子下面@好友来看，等于把已有内容转化为一次定向社交行为，不需要自己重新发帖。

**"提醒谁看"与"@"的区别**：

| | 提醒谁看 | @艾特 |
|--|---------|-------|
| 正文中展示 | 不展示 | 蓝色@文字 |
| 谁知道 | 只有被提到的人 | 所有人 |
| 心理负担 | 轻 | 重 |
| 使用频次 | 高（随手用） | 中低（慎重用） |
| 社交信号 | "我悄悄想让你看到" | "我当着大家面叫你来看" |

两者不互斥，同一条帖子可以既"提醒"了A，又在正文里@了B。

---

### 第三层：限制可见范围

**解决的问题**：用户不敢发。

让用户在发帖时控制"谁能看到这条内容"。选了指定好友可见后，只有被圈定的人能在feed中刷到这条帖子，其他人完全不可见。

对应的用户场景：
- 打完一局想吐槽但不想被陌生人看到
- 发一些只跟特定游戏相关的内容，想控制受众范围
- 有些玩梗或私密内容只想让队友看到
- 对"我的游戏记录被谁看到"有顾虑，给他一个明确的控制权能打消授权焦虑

这是一个被动的隐私保障。不产生通知，不改变帖子内容形态，只缩小了受众范围。跟前两层的关系是：它可以和"提醒谁看"或"@"组合使用——一条帖子既可以限定范围，又可以确保范围内的人一定看到。

## 三、完整功能清单

### 发帖侧

| 能力 | 入口位置 | 效果 |
|------|----------|------|
| 提醒谁看 | 发帖界面工具栏，📨 icon独立入口 | 被提醒的人feed中优先展示+"提到了我"标记，不改变正文 |
| @艾特 | 发帖界面工具栏，@ icon独立入口，以及正文中打@触发 | 蓝色@文字显示在正文中，被@的人收到强通知 |
| 限制可见范围 | 发帖界面"选择可见范围"弹窗中新增"指定好友可见" | 帖子仅对选定的人可见 |
| 标签化 | 选人页面选完后弹出"保存为标签" | 下次可一键选择同一批人，降低重复操作成本 |

### 阅读侧

| 能力 | 展示位置 | 效果 |
|------|----------|------|
| 朋友Tab头像提示 | Tab栏，用发帖人头像替代"朋友"文字 | 当有人@或提到自己时，能够显性提示用户 |
| "提到了我"标记 | 帖子卡片下方，浅灰色文字 | 轻量提示，不打断阅读流 |
| 优先排序 | 朋友Tab信息流 | 被提到/@的帖子即时排在最前 |
| 消息通知 | 消息页面社交互动流 | 区分"在动态中提到了你"和"分享了一条动态给你" |
| 角标 | 个人中心铃铛icon | 数字角标提醒未读 |

### 互动侧（评论区）

| 能力 | 触发方式 | 效果 |
|------|----------|------|
| 评论区@好友 | 点击帖子右下角"••"→ 评论 → 输入框旁@按钮 | 邀请好友来看这条帖子，被@的人收到"XX在评论中提到了你"的通知 |
| 正文手动@ | 在评论文本中输入@字符 | 弹出选人浮层，选择后插入蓝色@文字 |

## 四、对社区生态的预期价值

1. **降低发帖门槛**：有了范围控制，用户更敢发。尤其是那些一直不愿意授权的沉默用户——知道只有队友能看到，发一些随意的内容就没那么大心理压力。

2. **提升互动确定性**："提醒谁看"让发出的内容一定有人看到、有人回应。有回应就有正反馈，有正反馈就会继续发。这个循环一旦转起来，活跃度是自然增长的。

3. **召回沉默用户**：当有人@你或提到你，你就有了一个具体的理由回到社区来看看。对于那些装了游戏但很少打开社区的用户，这是一个低成本的唤醒手段。

4. **培养平台粘性**：当用户形成"发东西时顺手提醒几个人来看"的习惯，他的社交行为就绑定在了这个平台上。这些小型的、持续的社交互动，比偶尔发一条公开帖子的留存效果强得多。

5. **差异化竞争**：TapTap、小黑盒等平台没有微信的社交关系链，做不了基于真实人际关系的定向触达。这是微信游戏社区独有的结构性优势，值得深入挖掘。
