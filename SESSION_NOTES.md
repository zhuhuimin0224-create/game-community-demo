## 功能三层架构（已确定）

### 第一层：提醒谁看（轻量定向触达）
- 入口：发帖工具栏，📨 icon + "提醒谁看"
- 效果：不在正文显示，被提醒的人feed中"提到了我"浅灰色标记
- 选人页：搜索 + 导入标签 + "最近曾一起玩" + 好友列表（游戏社区互关优先）

### 第二层：@艾特（显性社交行为）
- 入口：发帖工具栏，@ icon + "用户"；评论区@按钮
- 效果：蓝色@文字显示在正文中，被@的人收到强通知
- 选人页：同上布局

### 第三层：限制可见范围
- 入口：发帖"选择可见范围"弹窗中"指定好友可见"
- 效果：帖子仅对选定的人可见
- 选人页：同上布局（标题"谁可以看"）

## 两者关键区别
- "提醒谁看"：隐性，只有被提到的人知道，轻量，高频
- "@"：显性，所有人看到蓝色@文字，重度，慎重用

## 重要产品决策记录

### 需求本质（不是"私密动态"）
- 帖子不一定是私密的，也可以是公开的
- "指定来看"不等于"只有这些人能看"，而是"这些人一定要看到"
- 核心价值是"定向触达"，不是"隐私保护"（隐私保护是附加项）

### 三件事的关系
1. 限制范围 = 被动隐私设定（谁有权看到）
2. 提醒谁看 = 轻量触达（悄悄让对方注意到）
3. @ = 显性触达（公开喊话）

### 为什么主推"提醒谁看"
- 心理负担低→使用频次高→粘性更强
- 不暴露社交关系（隐私友好）
- 跟微信朋友圈"提醒谁看"心智一致
- @作为社区基础能力自然存在，不需要重点推

### 发帖工具栏最终结构（从上到下）
1. # 话题
2. ◇ 投票
3. 📨 提醒谁看（右侧箭头左边展示已选人头像）
4. @ 用户
5. 📝 写长图文

### 键盘态横条工具栏
