## 功能三层架构（已确定）

### 第一层：提醒谁看（轻量定向触达）
- 入口：发帖工具栏，📨 icon + "提醒谁看"
- 效果：不在正文显示，被提醒的人feed中"提到了我"浅灰色标记
- 选人页：搜索 + 导入标签 + "最近曾一起玩" + 好友列表（游戏社区互关优先）

### 第二层：@艾特（显性社交行为）
- 入口：发帖工具栏，@ icon + "用户"；评论区@按钮
- 效果：蓝色@文字显示在正文中，被@的人收到强通知
- 选人页：同上布局

### 第三层：限制可见范围
- 入口：发帖"选择可见范围"弹窗中"指定好友可见"
- 效果：帖子仅对选定的人可见
- 选人页：同上布局（标题"谁可以看"）

## 两者关键区别
- "提醒谁看"：隐性，只有被提到的人知道，轻量，高频
- "@"：显性，所有人看到蓝色@文字，重度，慎重用

## 5/21 更新记录

### 与带教对齐后的方向修正

**核心目标变了：**
- 不是"让更多人来发帖"，而是"已经在发帖的人怎么通过内容创造更多价值"
- 通过已发帖的活跃用户触达更多人、带动更多人来社区并产生消费

**功能优先级调整：**
- 第一版不需要同时做"提醒谁看"和"@"两个功能——用户分不清区别
- 先聚焦一个做好：邀请别人来看
- 家慧留的两个核心问题：
  1. 怎么让"邀请人来看"做得更好玩？
  2. 被邀请来的人进来之后怎么承接？让他愿意看+看更多

**家慧确认的基本能力：**
- 邀请后一定给对方发红点
- 对方进来后邀请内容一定放首条
- 不要从消息/互动路径进去

**家慧补充的页面结构：**
- 圈子页：所有可被分发内容的推荐流，顶部有异形卡展示个人游戏动态（五杀、连胜、AI战报）
- 朋友页：时间+亲密度排序，以人的维度展示好友昨天一天的动态

### 新做的Demo页面（invite-flow.html）

**在线链接：https://zhuhuimin0224-create.github.io/game-community-demo/invite-flow.html**

包含以下场景：

**场景1：发帖后邀请（发帖承接页）**
- 个人主页 → 发帖 → 圈子页
- Toast浮层（10秒消失）：发布成功🎉 + 邀请好友来看 + 2个头像 + 加号 + 飞机发送
- 帖子下方三个点菜单 → 底部弹窗（转发给/转发朋友/分享朋友圈/邀请好友来看/删除）

**场景2：异形卡邀请围观**
- 圈子页顶部：圈子图标横栏 → 异形卡（深色渐变，游戏高光数据）
- "与shynew、kobe一起 · 邀请围观 ∨"
- 点击展开：5个好友头像 + 加号 + 飞机发送

**场景3：被邀请人看帖子（被邀看帖）**
- 圈子页顶部邀请内容卡（头像+邀请你来看+帖子原文+游戏卡）
- 底部三个互动入口：赞/评论/表情
- 第一层接续：同游戏热帖
- 第二层：好友胶囊横条（XX在玩王者）
- 第三层：泛推荐流

**场景4：被邀请人围观高光（被邀围观）**
- 圈子页顶部深色高光卡（Venus五杀了！邀请你围观）
- 投票互动："这波谁carry的？" → Venus/团队配合/对面太菜
- 第一层：你在这个游戏的数据（段位、胜率）
- 第二层：好友胶囊（XX今日3连胜）
- 第三层：同游戏热帖

### 被邀请人承接逻辑（关联度递减）

1. 邀请内容本身 + 轻互动入口（做一个动作）
2. 同游戏/同主题热帖（强关联）
3. 好友也在这里活跃（人的维度）
4. 泛推荐流（最后兜底）
