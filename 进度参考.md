### 邀请触发的两个场景

**场景1：用户主动发帖后**
- 发帖完成 → 跳转圈子页 → 帖子置顶
- Toast浮层（10秒自动消失）：发布成功🎉 + "邀请好友来看，获得更多流量助推"
- Toast右侧：2个常玩好友头像（点选）+ 加号（跳选人页）+ 飞机（发送邀请）
- 帖子三点菜单也可邀请

**场景2：游戏高光出现时**
- 圈子页顶部异形卡展示高光数据（五杀/连胜/MVP）
- 底部"与XX、XX一起 · 邀请围观 ∨"
- 展开：5个好友头像 + 加号 + 飞机发送

### 被邀请人的承接逻辑

**触达：** 红点 → 进入社区 → 圈子页顶部首条

**内容层次（关联度递减）：**
1. 邀请内容本身 + 轻互动入口（赞/评论/表情/投票）
2. 同游戏/同主题热帖
3. "好友也在这里"胶囊横条
4. 泛推荐流

## 带教反馈的核心要点

1. 优先为已发帖的人服务，不是为沉默用户降门槛
2. 第一版不需要同时上"提醒谁看"和"@"两个功能
3. 邀请功能要做得"更好玩"——区别于其他社区的普通@
4. 被邀请来的人要"愿意看+看更多"
5. 不要从消息路径进入
6. 6个idea还需要做demo
